# PlaceMux — Job Ranking System
Phase 2 | Task 3 | Search & Discovery

Skill-based ranking for the PlaceMux internship marketplace.
Ranks jobs for students and ranks candidates for companies, with a score and explanation on every result.

---

## How it works

Scoring is a blend of three things:
- **Skill overlap** — what fraction of required skills the candidate actually has (baseline)
- **TF-IDF similarity** — catches partial matches and related skills
- **Communication score** — small weight from the profile data

Weights: overlap 50%, tfidf 40%, comm 10%. An extra 2% bump is added if the candidate meets the experience requirement.

## API routes

| Method | Route | What it does |
|--------|-------|-------------|
| GET | `/students` | list all students |
| GET | `/students/{id}` | get one student |
| GET | `/jobs` | list all jobs |
| GET | `/jobs/{id}` | get one job |
| POST | `/rank/jobs` | rank jobs for a student |
| POST | `/rank/candidates` | rank candidates for a job |
| GET | `/metrics/jobs/{student_id}` | precision/recall/FPR for that student |
| GET | `/metrics/candidates/{job_id}` | precision/recall/FPR for that job |

### Rank jobs by student_id

```bash
curl -X POST http://localhost:8000/rank/jobs \
  -H "Content-Type: application/json" \
  -d '{"student_id": 1, "top_n": 5}'
```

### Rank jobs by skills (ad-hoc)

```bash
curl -X POST http://localhost:8000/rank/jobs \
  -H "Content-Type: application/json" \
  -d '{"name": "Vedant", "skills": "Python,Machine Learning,SQL", "top_n": 5}'
```

### Rank candidates for a job

```bash
curl -X POST http://localhost:8000/rank/candidates \
  -H "Content-Type: application/json" \
  -d '{"job_id": 101, "top_n": 5}'
```

## Metrics explanation

- **Precision** — of the top-K results shown, how many were actually relevant (score >= 60%)
- **Recall** — of all relevant results in the pool, how many ended up in top-K
- **False Positive Rate** — irrelevant results that slipped into top-K

The evaluate.py script also runs a random-shuffle baseline so you can see how much the ranker actually helps vs pure luck.

---

## Edge cases

- Skills with special chars (C++, C#, .NET) — handled in TF-IDF tokenizer
- Empty or missing skill strings — normalize_skills returns an empty set, no crash
- Student with 0 matching skills — still gets a score, just a low one
- Experience threshold not met — no hard cutoff, just no exp bump
